## Критика и слабые зоны

### Анализ и оценка
- **Score-механика хрупкая** — бонусы добавляются к 100, штрафы вычитаются. `100 - penalty + bonus` может дать >100 или странные результаты. Бонусы за «нет циклов» определяются по подстроке в `issue.title` — при смене формулировок сломается.
- **Не все параметры учитываются** — `frequency`, `synchronous` на связях частично используются. Многие параметры нод (requestRate, hitRate, reliability) почти не влияют на правила.
- **Нет весов правил** — все issues одного severity штрафуются одинаково, хотя важность правил разная.
- **Метрики не объясняются** — откуда «Сложность frontend 8.0», как считается «Нагрузка на API» — пользователь не понимает.

### Редактор
- **Нет Undo/Redo** — критично для редактора схем. Один неверный клик — потерю не откатить.
- **Нет multi-select** — нельзя выделить группу нод и перенести их вместе.
- **allowedEdgeKinds неполный** — `database` и `cache` с пустыми `{}`, т.е. «из них нельзя вести связи». При этом импорт JSON с любой связью проходит — клиент не блокирует недопустимые графы.
- **Добавление нод только через sidebar** — нельзя drag-and-drop, элемент появляется в случайном месте.
- **Нет поиска/фильтра** — при 20+ нодах сложно найти нужную.
- **Нет экспорта в изображение** — только JSON. Нельзя вставить схему в док/презентацию.

### UX и доступность
- **Слабая мобильная поддержка** — canvas-редактор на телефоне почти неюзабелен.
- **Нет горячих клавиш** в редакторе — Delete для удаления, Escape для снятия выбора и т.п.
- **Accessibility** — focus management, aria-атрибуты, навигация с клавиатуры для canvas.

### Инфраструктура
- **Нет unit-тестов** на правила анализа — при изменении формул легко сломать логику.
- **Версионирование схемы** — при смене правил старые результаты могут стать некорректными.
- **Один пользователь** — нет совместного редактирования, истории изменений.

### Контент
- **Issues без приоритизации** — «что исправить первым» непонятно.
- **Нет шаблонов** — «E-commerce», «Блог», «Микросервисы» для быстрого старта.
- **Рекомендации общие** — мало конкретики «сделай X вместо Y».

- [ ] **Корневой элемент «Система»** *(большая задача)*:

  - У корня свои опции (название системы, окружение, масштаб и т.п.)
  - Один такой элемент на схеме — валидация «уже есть корень»
- [ ] **Шаблоны архитектур** — пресеты «Стандартный CRUD», «Микросервисы», «С событийной шиной» для быстрого старта
- [ ] **Поиск по графу** — поиск нод по названию/типу, подсветка результата
- [ ] **Zoom to fit / Center** — удобная навигация по большим схемам (частично есть через React Flow)
- [ ] **Проверка при импорте** — валидация связей по allowedEdgeKinds, предупреждения о «невозможных» ребрах
- [ ] **Unit-тесты для правил** — каждый rule — отдельный тест с fixture-графом
- [ ] **Версия правил в ответе** — `rulesVersion` в AnalysisResult для совместимости со старыми клиентами
