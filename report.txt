  ✅ МОНОЛИТНЫЙ ANALYSIS SERVICE — РЕШЕНО (TASK-001)
     Рефакторинг завершён. Модульная архитектура:
     interfaces/ → engine/ (RuleEngine, GraphContext, MetricsCalc, ScoreCalc) → rules/structural/
     4 правила вынесены в отдельные классы. Пороги в analysis.config.ts.

  ⚠  ОТСУТСТВИЕ ВАЛИДАЦИИ НА БЭКЕ
     DTO (architecture-graph.dto.ts) не имеют декораторов class-validator.
     Любой мусор проходит в analyze(). Потенциально может крашить сервер.
     Файл: server/src/analysis/dto/architecture-graph.dto.ts

  ⚠  ТИПЫ НЕ СИНХРОНИЗИРОВАНЫ
     Frontend (shared/model/types.ts) и backend (dto + analysis.service.ts)
     определяют одни и те же интерфейсы независимо. При расхождении — баги.
     TASK-030 — синхронизация.

  ✅ ВИЗУАЛЬНОЕ РАЗЛИЧИЕ НОД — РЕШЕНО (TASK-012, TASK-013)
     Кастомный React Flow node-тип «architecture» с цветами по слоям:
       Frontend — синий, Backend — зелёный, Data — оранжевый.
     Каждый kind имеет иконку-эмодзи (📄🧩📦🌐⚙️🗄️⚡☁️).
     Связи визуально различаются: цвета, пунктир, анимация для subscribes/emits.

  ⚠  EDGE PARAMETERS НЕ ИСПОЛЬЗУЮТСЯ
     Поля frequency и synchronous определены в типе GraphEdge,
     но не учитываются ни в UI edge-properties, ни в анализе.
     Связи TASK-023 → TASK-007 (L06 sync chain latency).

  ⚠  НЕТ ОБРАБОТКИ ОШИБОК ЗАПРОСА АНАЛИЗА
     При недоступном бэкенде или ошибке — ничего не происходит.
     Нет loading state, нет error state. TASK-027.

  ⚠  ПРЯМОЙ ДОСТУП К /analysis
     Если пользователь перейдёт на /analysis без предварительного анализа,
     страница будет пустой. Нужен guard/redirect. TASK-026.

    analysis/
      interfaces/    — AnalysisRule, GraphContext, AnalysisIssue, AnalysisResult
      engine/        — RuleEngine, GraphContext builder, Metrics/Score calculators
      rules/
        structural/  — S01-S10 (10 правил)
        pattern/     — P01-P08 (8 правил)
        load/        — L01-L06 (6 правил)
      analysis.config.ts — все пороговые значения (structural, pattern, load, scoring)
      analysis.service.ts — тонкий оркестратор
      analysis.controller.ts, dto/
    app.*            — root module
